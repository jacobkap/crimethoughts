Most code you write for research is a waste of time (because someone else has already done it)

2022-05-02

It is very rare in criminology research to use quantitative data that no one has ever used before.1 It does happen. Primarily in surveys that the authors wrote themselves - including modifying past surveys - but also through collecting administrative data yourselves such as through data sharing agreements or FOIA requests. The majority of papers, however, rely on data the other people have used before. Often thousands of times before since many criminology papers use publicly available administrative data from government agencies such as the FBI, BJS, or local police departments.

So how does the data cleaning part of the papers that use these common datasets go? Exactly the same as if it were completely original data that no one touched before. Let’s use the FBI’s Uniform Crime Reporting Program Data as an example. If you wanted to use this data you’d download it from openICPSR and write some code to clean it. You’d grab just the rows and columns you want, aggregate it to a certain unit, check for outliers, make some graphs, and so on for whatever your use case it. Now imagine I wanted to do basically the same study but with a tiny variation. My data cleaning process would be nearly identical to yours. I’d be spending the time working with the data even though I’m mostly doing the same work you already did.

There are two major issues with people writing their own code when using data that other people have used. First, it’s an enormous waste of time. Most criminology research is not that complicated. It’s a lot of correlational studies analyzing a small number of datasets - often only a single dataset. Even studies that try to measure causal relationships often are relatively simply to think about and to analyze, using difference-in-difference approaches or natural experiments. What takes a lot of time on the analysis side (so excluding writing up the paper) is working with the data. Crime data is messy. It’s usually rather miserable to work with.2 Criminology papers tend to take forever to do because, at least in part, the data available is usually hard to work with and the people working with it are not good at programming. So researchers - or to be more specific, their graduate students - take up so much time to because the data they use is not very friendly, even to the most experienced programmer. So take the amount of time it takes to clean up data for a project and then multiply it by the number of studies that use this data. This is the amount of time wasted by people redoing work that other people have already done. This is likely dozens or hundreds of hours - if not more - per person each year spent doing work that other people have already done.

And who is doing this work? For many studies it’s graduate students, the least qualified group to be doing research.3 This leads to the second major issue, having many people do the same code makes it more likely that someone will make a mistake. If 10 people write code to clean the same dataset there is a good chance that someone will make a mistake in the code. If 100 people do, its guaranteed that someone will. Considering how many researchers are not formally trained in programming, there’s probably a lot of programming mistakes in papers.

The most obvious solution to this is to require - or at least encourage - people to make their code public when they publish a paper. If, in the example above, your code was public I could have saved a lot of time by reusing your code instead of doing it myself. I do not think this is a useful solution to this problem.4 And it could actually be worse than people writing their own code every time. I believe this for two reasons: 1) most code is written for a specific use case so won’t actually solve the problem of data being messy and 2) many researchers are bad programmers, and repeating bad code is dangerous. This solution is also practically more difficult than the one I’ll propose below, primarily due to the perverse incentives to hoard data cleaning code.

The first issue is a big one: for most research you’ll only use a tiny part of the data. For example, if you’re using police crime data you’ll usually use only a small number of crimes - often murder or index crimes - and ignore the rest. So you’ll probably look only at a relatively small number of rows and columns from the entire dataset. If I copy your code and only look at the same rows and columns then I’ll be fine. But as soon as I’m interested in other parts of the data your code may not apply. It’s a bit like extrapolating your regression results beyond the bounds of your data. If, for example, the variables you looked at had no missing data at all your code wouldn’t do anything to handle missingness. If I rely on your code but use it for variables with missing data I’d have an issue as I’d do nothing to handle that missing data. Given how complex crime data is - and how many data issues there are - you’ll need to carefully check every variable to make sure there aren’t any issues.

The second issue is that most researchers are not programmers. That is, they’re not formally trained to write code and tend to spend little time writing it other than the bare necessity for their specific project. This means that their code is often slower, much longer, and more fragile than necessary. By fragile I mean that it’s prone to breaking if something is changed - for example hard coding variable names instead of writing functions. This makes it hard to reuse the code because you’ll need to change all of these hard coded variables to new variables. Miss even one - and I’ve certainly done this myself when reusing old code - and you’ll cause a major error such as selecting the wrong variable. This kind of code is also the most prone to errors. Even the most experienced programmers who have the time to write careful code and have other people review the code can make errors so having people inexperienced and not focused on writing good code makes it more likely that there will be an error. And with that error the results of the paper will be wrong.

The danger in using code released by paper authors as a solution to replicating data cleaning work is that an error in the original code will be likely be repeated by people using this code. Having the code be public is a way to find these errors, but I doubt that people will do this. Some will but these are the people who are likely pretty experienced programmers anyways so will have the least need to use other people’s code for their research. And there’s a lot of evidence that people often do not do their due diligence when checking on the data they’re using or the papers they’re citing. Consider, for example, how many people use county-level UCR data even now two decades after Maltz and Targonski warned against it or that retracted papers continue to be cited. So I have little faith that they’re doing a comprehensive examination of the code they find to ensure that there are no errors.

Currently there is little work being done by academics to improve the access of data other than the relatively little amount of replication data that is released.5 There are exceptions to this, such as Matt Ashby’s Crime Open Database or my own work with most FBI data.6 The main reason for this, I believe, is because criminology strongly disincentivizes both data work and the sharing of data. The currency of criminology is in published papers. You get an academic job by writing lots of papers - better quality is a plus, but quantity seems more important.

Good data work - which includes carefully checking every variable and the overall data, and producing documentation about the data - takes a long time. And while it’s certainly helpful for research this is work that doesn’t directly translate to papers so is infrequently done. Given how important papers are, taking the time to do this work is a substantial amount of time not working on papers. And when people release the data after all of this work anyone who uses the data gets to shortcut that work. If you spend 6 months on some data and release it publicly, I can use it immediately, saving me 6 months of work. And I could have also spent those 6 months working on papers that get me a job (or tenure, grants, prestige etc.). So if someone does work on data for a while, they have a strong career incentive to keep it to themselves.

For the data work that academics do, that’s largely done on their own time as a result of a desire to improve crime data or as ancillary to a paper. And usually they’re the only ones working on that particular dataset. If they stop for whatever reason, that data likely will not be updated again. As an example of this, in 2020 David Abrams, a law professor at Penn, started a website tracking near real-time crime data from over two dozen large US cities.7 This site allowed users to visualize crime data from a number of different cities and see how it changed over time. That project led to a published paper and while the site is still available it has not been updated since 2021.

This isn’t to say that there is no good crime data work being done, just that it’s rarely academics doing it. It’s mostly think tanks (which do research, but I consider them outside of academia), news organizations, advocacy groups, and the government doing this work. To name but a few there’s the excellent Prison Population Forecaster from the Urban Institute and a murder data dashboard tracking near real-time murder data from a number of large cities by Jeff Asher and Ben Horwitz. There’s also an explosion of data journalism allowing users to interact with crime data without any programming knowledge (see here for one example). Government agencies also provide useful crime data tools like the (flawed) Crime Data Explorer from the FBI and the National Crime Victimization Data Dashboard by the BJS. These projects are great and are doing a tremendous amount of good for our access and understanding of crime data. Academics have a lot to offer in terms of understanding the data and do conduct research on assessing how good the data is.8 By limiting ourselves primarily to conducting research instead of improving crime data - which is in essence improving the infrastructure for all future research - we’re doing a disservice to our fellow academics and field of crime research at-large.

There are cases where academics receive funding to focus on good crime data collection. An instructive case of what happens when money and focus is spent on improving crime data is the Jail Data Initiative, part of the Public Safety Lab at NYU. This project is “scraping daily county jail rosters and criminal case records in over 1,000 counties” and has quickly created that best source of jail data that I’m aware of. There’s very little data available about jails, particularly about individual people in jails, so this is filling a huge gap in available data. Importantly, this is not work that (as far as I’m aware) requires extensive data sharing agreements with agencies or special relationships with people who have the data. Everything they’re scraping is public. It just requires a huge amount of work to collect this data, especially to collect it every day given how frequently websites change (and thus the scraping no long works). Their work has been used in several reports (see the link earlier for a list of them) as well as academic articles. This project is funded by private donors, including Arnold Ventures, the Chan Zuckerberg Initiative, and Pew Charitable Trusts, so funders are willing to fund projects to improve criminal justice data quality and access.9

Work like the Jail Data Initiative is an excellent example of what I’m proposing. My solution to the problem of bad data and the massive waste of effort for people cleaning the same data: make crime data easier to use. This sounds pretty dumb (or at least way too obvious) so let me explain. If we make data cleaner at the start - that is, from the point where researchers download it - then everyone using it will have less work to do and can focus their attention to analyzing the question they have.10 Cleaner data means users will need to write less code which then means fewer opportunities to make mistakes.

Cleaning up data is a good ideal but how do I propose actually doing it? Public and private funders should pay people to make crime data easier to use. This includes basic things like converting data into different formats - for example from ASCII files to R or Stata file formats. Lots of useful data is hidden in tables on government websites or PDFs so it can also include scraping that data and making it available in a machine-readable format. If this proposal sounds a lot like what ICPSR or NACJD does, well you’re right. It is partially advocating to do more of what they do - and go beyond it.

They take administrative datasets and make them publicly available while providing a useful codebook detailing broad information about the data and about each variable. What I’m proposing goes further than what ICPSR/NACJD do. They are what I consider a neutral party for data. They get the data and release it without judgement or guidance on how to use it well. This is a good thing since without their hard work we’d have far less data available. What I think we also need is a sort of positive party for data.11 This involves collecting data from hard to use, often disparate places, like the Jail Data Initiative does. Lots of crime data is like this, in many different agency’s websites or PDFs, and could be gathered, but simply requires the (often ample) resources to do so.

Then what I think is truly key to this is to create extensive documentation not just about what is in the data but how to use the data. To give an example from my own book on using NIBRS data, the data tells you which hour each crime happened during. In most codebooks this would be the end of the description of that variable. But NIBRS has a weird feature where midnight-1am and noon-1pm are, by far, the most common hours for a crime to happen. This is, in my opinion as someone knowledgeable about the data (and crime data in general), not true but is merely a quirk of the data.12 This is something that each individual user could potentially discover on their own but is far more likely to be found by someone whose intent is to try to understand the data and all of its issues. While I believe that having people focus on how to use the data will illuminate a lot of these problems, they won’t be able to understand every part of the data, especially for big datasets with many hundreds or thousands of variables. And for each individual variable they may spend less time on it than someone who uses only a small part of the data in their research. Many researchers approach using crime data like using a flashlight in a dark room. They’ll be able to see the specific parts of the room they’re shining the light on (the parts of the data they use in the study) but have large dark spots for unlit areas. For someone who focuses on the data to try to understand it entirely, not for a specific project, it’s like standing in the room waiting for their eyes to adjust to the darkness. It takes much longer, and they’ll likely bump into furniture a lot more than with a flashlight (learn about problems in the data by encountering them) but they’ll end up with a much better understanding of the whole room (the data).

For funders who want to increase research in a topic, paying to improve data quality and access is probably the most cost-effective thing to do. It won’t work for data that’s private or requires special access, but for public but hard to use data it can be a game changer. Most researchers won’t spend dozens of hours (or hundreds, or thousands) trying to get data. But they will download data that someone else has already spent that time on and use it in their research. Most funding that I’ve seen (from both private and public donors) are research-specific, funding researchers to study a certain topic. Funding for data would do the same, but indirectly. It’d make research easier and let the strong paper-dominated incentives of criminology do the rest. Throwing money at this problem, however, is not the same as fixing it. And there are some dangers.

A likely negative outcome of funding data projects is that people who are not qualified to work on the data will be funded, and any mistake they make will be compounded in all research that uses the data. Since I’ve spent much of this post arguing that normal academics writing code isn’t a good solution, partially due to them not being programmers, funding should go to people whose focus is on a particular dataset and who are good programmers. Though this doesn’t necessarily have to be the same person, a team of people is often better. As an added check, all code used to gather, clean, or otherwise interact with the data should also be made publicly available. I want to be clear that I’m not proposing hiring a bunch of people with no understanding of the data. Crime data is tricky because of the nuances of what it means and how it’s created, not just because the dataset is hard to use. This should be criminologists (broadly defined, including people from other fields such as econ, political science, and others) who have programming skills and experience with this particular data to do the work.

Another important tradeoff to making the data easier to use is that when you lower the bar to use the data and you also lower the bar of who will use it. This can lead to people conducting research with the data who don’t spend the time to actually understand the data and the inherent problems with it. See, for example, all of the papers that use county-level UCR data. Crime data is very tricky. There are importance nuances to understanding and believing the data that take knowledge of how, for example, police operate (and thus input the data) and how the criminal justice system works to truly use the data properly. Make it too easy to use and you invite people using the data badly. Given (my hope that) criminology data actually affects policies, bad research can affect people’s lives in important ways.13 And I use the word “invite” specifically. People who make this data available and easier to use do have a responsibility over what people use the data for, even though they’ll likely have no actual control over how it’s used.

To reiterate this post, most crime data is hard to use due to important nuances in the data and complex data formats. Most researchers are not programmers so have limited ability to use complicated data. A solution to this is for public and private funders to pay for projects that improve crime data access and usability, including producing documentation on how to use the data properly. It’s just like the old saying: if you build a machine-readable format of easy to use data and extensive documentation, they will do come and research on it.
